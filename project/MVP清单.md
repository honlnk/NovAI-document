# NovAI MVP 清单

最后更新：2026-04-01

## MVP 目标

MVP 的目标不是做出完整的小说创作平台，而是验证 NovAI 最核心的产品假设：

1. 用户是否愿意以“本地文件项目”的方式进行小说创作
2. AI 是否能够围绕章节文件和要素文件稳定工作
3. 文件系统 + RAG 的上下文管理方式，是否比传统聊天记忆更适合长篇小说创作

因此，MVP 只聚焦最小闭环：

`创建项目 -> 配置模型 -> 输入指令 -> 生成章节 -> 写入文件 -> 提取要素 -> 下次生成可复用要素`

## MVP 范围

## 必须完成

### 1. 项目创建与打开

- 创建小说项目
- 初始化标准目录结构
- 打开已有项目
- 读取项目配置和基础文件

这是所有后续能力的入口，没有这一步，整个产品无法成立。

### 2. 文件树与文件预览

- 展示 `chapters/`、`elements/`、`prompts/` 目录
- 点击文件后可查看内容
- 文件变更后可刷新展示结果

这是“文件为中心”的基础体验，没有文件树就无法体现 NovAI 与普通聊天工具的区别。

### 3. 模型配置

- 配置 LLM API
- 配置 Embedding API
- 支持测试连接
- 保存到 `novel.config.json`

这是 AI 能力运行的必要前提。

### 4. 内容生成主链路

- 输入自然语言创作指令
- 读取 SYSTEM 提示词
- 读取最近 N 章内容
- 调用 LLM 生成章节
- 以流式方式展示生成过程
- 将结果写入 `chapters/` 中的 Markdown 文件

这是 MVP 的核心，没有这一步就还不是 NovAI。

### 5. 要素提取

- 从新章节中提取人物、地点、情节、时间线等要素
- 将提取结果写入 `elements/` 子目录
- 定义最小可用的要素文件格式

这是 NovAI 能进入“长篇可持续创作”模式的起点。

### 6. 向量化与粗检索

- 对要素生成 Embedding
- 建立 Orama 索引
- 在下一次生成时，根据用户指令检索相关要素
- 将检索结果拼入生成上下文

这一步完成后，NovAI 才和普通“AI 帮你续写”工具拉开差异。

## 可以延后

以下内容很重要，但不建议阻塞 MVP：

- AI 精筛选（两阶段检索中的第二阶段）
- Git 自动提交与版本管理界面
- 校对模块
- 章节整理模块
- 提示词调教对话工作流
- 高级设置页
- 外部文件变更监听优化
- 多种复杂视图切换

这些内容适合在 MVP 核心链路跑通之后继续补强。

## 第一步先做什么

当前建议的第一步不是直接接 AI，而是先把“项目工作台骨架”搭起来。

原因：

- 没有项目创建与打开能力，AI 无处写文件
- 没有文件树和预览，用户无法感知“AI 正在操作小说工程”
- 没有基础页面骨架，后面的配置、生成、预览都无从承载

所以建议第一批任务按这个顺序推进：

1. 搭建路由与页面骨架
2. 完成项目创建 / 打开流程
3. 完成本地文件读取与文件树展示
4. 完成文件预览面板
5. 再接入模型配置和 AI 生成

## 建议的 MVP 执行顺序

### Step 1：工作台基础结构

- 首页
- 主工作区布局
- 设置页骨架
- 路由与状态管理

交付标准：

可以在界面中完成页面切换，并为后续功能预留稳定位置。

### Step 2：项目文件系统接入

- 新建项目
- 打开项目目录
- 初始化目录结构
- 扫描项目文件

交付标准：

用户能真实创建并打开一个本地小说项目。

### Step 3：文件树与预览

- 左侧文件树
- 点击文件打开
- 右侧 Markdown 预览

交付标准：

用户能浏览小说项目中的现有文件。

### Step 4：模型配置

- LLM 配置页
- Embedding 配置页
- 测试连接
- 配置持久化

交付标准：

用户能完成最基础的模型接入准备。

### Step 5：章节生成

- 输入指令
- 组装最小上下文
- 调用 LLM 流式生成
- 写入章节文件
- 预览生成内容

交付标准：

用户能通过一句自然语言指令生成一章内容并保存到本地。

### Step 6：要素提取与粗检索

- 提取要素
- 写入要素文件
- 建立 Embedding 索引
- 下次生成时检索相关要素

交付标准：

系统能开始复用历史内容，而不再只是一次性生成器。

## MVP 完成定义

满足以下条件时，可以认为 MVP 基本成立：

- 用户可以创建或打开一个本地小说项目
- 用户可以完成模型配置
- 用户可以输入一句创作指令生成章节
- 章节内容会写入本地 Markdown 文件
- 系统会从章节中提取要素并保存
- 下一次生成时可以利用已有要素辅助生成

## 备注

如果后续开发中发现某些“必须完成”项实际上会显著拖慢验证速度，可以调整顺序，但原则不变：

优先验证 NovAI 的核心闭环，而不是优先补齐所有外围能力。
