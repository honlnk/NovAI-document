# NovAI 项目总览

## 项目定位

NovAI 是一个面向长篇小说创作的 AI 工作台。

它的核心目标不是做一个普通聊天工具，而是让 AI 围绕本地文件系统、章节文件、故事要素和 RAG 检索来协助作者持续创作长篇小说。

## 当前阶段

当前处于：

`MVP 早期实现 / Agent Loop 骨架验证阶段`

现状特点：

- 产品规划、需求说明、UI 设计、技术架构文档已经初步成型
- 已完成项目目录初始化、配置读写、模型测试连接与流式生成的第一版实现
- 已完成第一版模型驱动的文件工具 Agent Loop
- 已实现项目级日志与最近项目恢复
- 当前以极简测试页为载体验证能力，不继续推进正式工作台 UI
- 需要继续把“工具调用 -> 写入确认 -> 要素提取 -> 检索复用”打通成稳定闭环

## MVP 目标

第一阶段重点验证以下闭环：

1. 创建并打开小说项目
2. 配置 LLM 与 Embedding
3. 通过自然语言指令驱动 Agent 读写项目文件
4. 将生成或修改结果写入本地 Markdown 文件
5. 从章节中提取要素并建立向量索引
6. 下一次生成时能够检索并利用已有要素

## 当前判断

当前项目最关键的不是补更多想法，而是尽快验证下面两件事：

1. “AI 直接操作小说文件”的交互链路是否顺畅
2. “长篇上下文靠文件 + RAG 管理”是否真的比聊天式记忆更稳定

如果这两点成立，NovAI 的产品方向就成立。

## 当前已落地能力

- 创建、打开与修复小说项目目录
- 初始化标准项目文件结构
- 读写 `novel.config.json`
- 测试 LLM / Embedding 配置
- 发起 LLM 流式生成
- 读取与保存 `prompts/system.md`
- 将生成结果写入 `chapters/`
- 在测试页中浏览项目文档原文
- 通过 `ReadFile / EditFile / CreateFile / RenameFile / DeleteFile / ListDirectory / FindFiles` 让模型主动查找、读取、创建、修改、重命名和删除项目文件
- 将工具结果回灌模型，支持多轮 Query Loop
- 将项目运行日志写入 `.novel/logs/agent.log.jsonl`
- 删除文件会先移入 `.novel/trash` 回收站，而不是永久删除
- 记住最后打开的项目，并在刷新后恢复
