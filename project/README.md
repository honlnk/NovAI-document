# 项目执行文档

这个目录用于维护 NovAI 的开发推进情况。

和产品规划、需求说明、技术架构这类“设计文档”不同，这里的文档聚焦于：

- 当前做到哪里
- 下一步做什么
- 每个阶段的目标是什么
- 重要决策和执行记录

## 文档清单

| 文档 | 作用 |
|:-----|:-----|
| [项目总览](项目总览.md) | 记录项目当前阶段、目标、整体状态 |
| [开发路线图](开发路线图.md) | 记录阶段划分、里程碑和优先级 |
| [MVP清单](MVP清单.md) | 记录第一阶段最小可用产品范围与执行顺序 |
| [当前进度](当前进度.md) | 记录已完成、进行中、待开始的事项 |
| [开发日志](开发日志.md) | 按时间记录关键进展、调整和说明 |
| [Claude Code 借鉴与映射设计](../ClaudeCode借鉴与映射设计.md) | 记录参考 Claude Code 后形成的产品映射与实现边界 |

## 维护原则

1. 每次完成一个阶段性功能后，更新“当前进度”。
2. 每次调整优先级或里程碑后，更新“开发路线图”。
3. 每次发生重要实现变化或范围调整后，补一条“开发日志”。
4. 如果代码现状与设计文档不一致，以“开发日志”说明原因，并在后续同步修正文档。

## 使用建议

- 设计类内容继续放在 `docs/` 根目录。
- 执行类内容统一放在 `docs/project/`。
- 后续如果任务变多，可以继续细分子文档，例如 `MVP清单.md`、`问题清单.md`、`版本计划.md`。
