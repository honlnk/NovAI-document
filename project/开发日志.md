# 开发日志

## 2026-04-29

### 完成第一版文件工具 Agent Loop

本次开发将会话主链路从“规则判断 + 单次 LLM 生成”推进为更接近 Claude Code 的工具型 Agent Loop。

当前已实现：

- 新增 `src/core/agent/` 分层
- 新增模型消息、工具调用、工具结果等 Agent 消息类型
- 支持 OpenAI-compatible `tools / tool_calls`
- 支持模型返回工具调用后执行工具，并把 `tool_result` 回灌给模型继续推理
- `query()` 默认最多循环 8 轮，避免无限调用
- Session Lab 已能展示用户消息、assistant 文本、工具调用与工具结果

当前最小文件工具集调整为：

- `ReadFile`
- `EditFile`
- `CreateFile`

其中 `EditFile` 采用精确 `oldText -> newText` 替换方式，`CreateFile` 在目标已存在时失败，`ReadFile` 支持行号输出与部分读取。

### 对齐 Claude Code 主链路认知

对照 Claude Code 后，当前确认 NovAI 的目标链路应逐步靠近：

```txt
用户输入
-> 提交处理
-> 上下文组装
-> query() agent loop
-> 模型流式响应
-> tool_calls
-> 工具执行
-> tool_result 回灌模型
-> 模型继续推理或结束
```

当前 NovAI 已完成中间核心链路：

```txt
用户输入
-> runChatTurn
-> query()
-> streamAgentCompletion
-> tool_calls
-> runAgentTools
-> tool_result
-> query() 下一轮
```

尚未完成的 Claude Code 式外围能力包括：

- `handlePromptSubmit / processUserInput / onQuery` 的清晰分层
- 写工具执行前的权限确认
- Query Guard / AbortController / 队列
- Stop hooks、成本统计、transcript 等运行指标

日志当前覆盖：

日志写入采用项目级串行队列，避免连续事件并发写入时覆盖或丢失。

日志时间现在同时记录：

- `time`：本地时间，带时区偏移，例如 `2026-04-29T15:40:00.000+08:00`
- `timeUtc`：UTC 时间
- `timezone`：浏览器时区，例如 `Asia/Shanghai`

### 新增最近项目恢复

新增 `src/core/project/recent-projects.ts`，使用 IndexedDB 保存最后一次打开的项目目录句柄。

当前行为：

- 创建、打开、修复项目成功后，保存为最近项目
- 页面加载时读取最近项目记录
- 如果目录权限仍然存在，则自动恢复项目
- 如果权限不存在，则显示“恢复上次项目”按钮，请求授权后恢复
- 支持“忘记上次项目”
- 恢复项目时只记录 `project_restored -> project_activated`，不再重复记录 `project_remembered`

### 当前验证结果

已人工验证：

- 打开项目会写入 `project_opened / project_remembered / project_activated`
- 刷新页面后可自动恢复最近项目
- 关闭项目会写入 `project_closed`
- 手动恢复会写入 `project_restored`，并区分 `requestedPermission`
- 忘记最近项目后刷新不会再自动恢复，也不会产生新日志

## 2026-04-08

### 明确 Claude Code 参考关系与 NovAI 的真实目标

在对照 `claude-code-sound` 之后，进一步明确：

- NovAI 不应继续被理解为“带聊天区的小说工作台”
- NovAI 更接近“面向小说创作者的、可视化的、对话驱动的 AI 智能体工具”
- 对话的职责是驱动 AI 通过工具读写文件，而不是单纯输出长文本

### 明确最小工具边界

当前确认 NovAI 第一阶段的最小工具集为：

- `Bash`
- `FileRead`
- `FileWrite`
- `FileEdit`
- `RagSearch`

其中：

- 前四项借鉴 Claude Code 的工具化思路
- `RagSearch` 是 NovAI 相比 Claude Code 的核心差异能力

### 明确不采用 CLAUDE.md 机制

虽然 Claude Code 会自动注入 `CLAUDE.md` 等规则文件，但 NovAI 不采用这套具体机制。

当前共识是：

- NovAI 仍然采用自己的提示词体系
- 会话上下文主要来自 `prompts/system.md` 与 `prompts/scenes/*.md`
- 提示词在 NovAI 中是正式业务数据，而不是隐式补丁规则

### 明确可视化默认目标机制

NovAI 后续 UI 与会话层要新增一条关键交互原则：

- 用户当前正在预览什么，AI 默认就优先把它视为当前操作目标

例如：

- 预览章节时，默认操作该章节
- 预览场景提示词时，默认操作该提示词
- 预览要素时，默认操作该要素文件

这和 Claude Code 依赖纯文本描述目标位置的方式不同，是 NovAI 面向非程序员创作者的重要改造方向。

### 新增映射设计文档

新增 `docs/ClaudeCode借鉴与映射设计.md`，用于系统记录：
