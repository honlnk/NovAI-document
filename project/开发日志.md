# 开发日志

## 2026-04-08

### 明确 Claude Code 参考关系与 NovAI 的真实目标

在对照 `claude-code-sound` 之后，进一步明确：

- NovAI 不应继续被理解为“带聊天区的小说工作台”
- NovAI 更接近“面向小说创作者的、可视化的、对话驱动的 AI 智能体工具”
- 对话的职责是驱动 AI 通过工具读写文件，而不是单纯输出长文本

### 明确最小工具边界

当前确认 NovAI 第一阶段的最小工具集为：

- `Bash`
- `FileRead`
- `FileWrite`
- `FileEdit`
- `RagSearch`

其中：

- 前四项借鉴 Claude Code 的工具化思路
- `RagSearch` 是 NovAI 相比 Claude Code 的核心差异能力

### 明确不采用 CLAUDE.md 机制

虽然 Claude Code 会自动注入 `CLAUDE.md` 等规则文件，但 NovAI 不采用这套具体机制。

当前共识是：

- NovAI 仍然采用自己的提示词体系
- 会话上下文主要来自 `prompts/system.md` 与 `prompts/scenes/*.md`
- 提示词在 NovAI 中是正式业务数据，而不是隐式补丁规则

### 明确可视化默认目标机制

NovAI 后续 UI 与会话层要新增一条关键交互原则：

- 用户当前正在预览什么，AI 默认就优先把它视为当前操作目标

例如：

- 预览章节时，默认操作该章节
- 预览场景提示词时，默认操作该提示词
- 预览要素时，默认操作该要素文件

这和 Claude Code 依赖纯文本描述目标位置的方式不同，是 NovAI 面向非程序员创作者的重要改造方向。

### 新增映射设计文档

新增 `docs/ClaudeCode借鉴与映射设计.md`，用于系统记录：

- 两个项目的关系
- 需要借鉴的核心逻辑
- 明确不照搬的复杂能力
- Claude Code 到 NovAI 的功能映射方式

## 2026-04-04

### 调整当前开发顺序

当前决定暂缓正式工作台 UI，先集中验证 AI 主链路。

因此，当前实现方式从“先搭工作台骨架，再接 AI”调整为：

`极简测试页 -> 项目读写 -> 模型测试连接 -> 流式生成 -> 文件写入`

### 新增测试页主链路

当前 `/test` 页面已经可以完成：

- 创建小说项目
- 打开已有目录
- 检测并修复不合法项目目录
- 读写 `novel.config.json`
- 测试 LLM / Embedding 连接
- 发起 LLM 流式生成
- 读取并保存 `prompts/system.md`
- 将生成结果写入 `chapters/`
- 浏览项目中的 Markdown / JSON / 文本文档原文

### 新增 AI 接口层

当前已补齐第一批 AI 基础模块：

- `core/llm/client`
- `core/embedding/client`
- `core/ai/shared`

并统一了测试页消费的流式事件协议。
