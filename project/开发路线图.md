# NovAI 开发路线图

## Phase 0：基础搭建

目标：完成项目代码基础盘，建立后续开发骨架。

计划内容：

- 搭建 Vue + Vite 基础结构
- 引入路由、状态管理和基础样式方案
- 建立首页、工作区、设置页的页面骨架
- 明确核心目录结构

## Phase 1：项目与文件系统

目标：让用户能创建、打开并浏览一个小说项目。

计划内容：

- 接入项目创建流程
- 初始化标准项目目录结构
- 接入本地文件夹访问能力
- 实现文件树浏览与文件打开预览

## Phase 2：模型配置与生成链路

目标：打通“输入指令 -> 生成章节 -> 写入文件”的最小闭环。

计划内容：

- 配置 LLM 与 Embedding
- 实现 API 连通性测试
- 构建 AI 对话输入区域
- 支持流式生成与写入章节文件
- 支持右侧预览生成结果

## Phase 3：要素抽取与 RAG

目标：让生成结果可以沉淀为可检索上下文。

计划内容：

- 设计要素文件格式
- 从章节提取人物、地点、情节等要素
- 建立 Embedding 与 Orama 索引
- 打通粗检索链路
- 为后续精筛选预留结构

## Phase 4：上下文增强与稳定性

目标：提升长篇创作场景下的质量和可持续性。

计划内容：

- 补齐两阶段检索中的 AI 精筛选
- 加入近期章节上下文拼装
- 完善错误处理与降级策略
- 优化生成中断、空结果、重试等边界情况

## Phase 5：辅助能力

目标：补齐工作台体验，提升实际可用性。

计划内容：

- Git 自动提交与历史查看
- 校对视图
- 章节整理视图
- 提示词调教工作流
- 设置页完善

## 当前建议优先级

当前建议先聚焦：

1. Phase 0
2. Phase 1
3. Phase 2

原因：

在真正验证“项目创建 + 文件操作 + AI 生成”之前，过早投入校对、版本管理、复杂交互，收益不高。
